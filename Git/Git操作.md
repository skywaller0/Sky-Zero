# Git 命令
## git init 
初始化一个Git仓库, 生成一个.git目录, 用于跟踪版本管理
## git add
将文件添加到暂存区, 也就是将文件添加到仓库中, 但是仍然需要commit才能提交到仓库
## git commit
将文件提交到仓库, -m后面输入的是本次提交的说明, 可以输入任意内容, 但是最好是有意义的, 这样可以从历史记录中方便的找到改动记录
## git status
查看仓库当前的状态, 可以看到哪些文件被修改过, 哪些文件被添加到暂存区, 哪些文件被提交到仓库
## git diff
查看文件修改的内容
## git log
查看历史记录, 可以查看到每次提交的commit id, 以及提交的说明
## git reset
回退版本, 也就是回退到上一个版本, HEAD表示当前版本, HEAD^表示上一个版本, HEAD^^表示上上一个版本, 也可以写成HEAD~100, 表示上100个版本
## git reflog
查看命令历史, 可以看到每一次的命令, 以及命令的commit id
## git checkout
撤销修改, 也就是将文件恢复到上一次git commit或者git add时的状态, 如果文件没有被添加到暂存区, 那么就是恢复到上一次git commit时的状态, 如果文件已经被添加到暂存区, 那么就是恢复到上一次git add时的状态
## git branch
查看分支, 当前分支前面会有一个*号
## git branch <name>
创建分支
## git checkout <name>
切换分支
## git merge <name>
合并某分支到当前分支
## git push origin <name>
将本地分支推送到远程分支


**Git其实就是有本地暂存区，有本地仓库，远程仓库，Git的全称分布式版本控制系统。工作时，首先切换到某个分支进行操作，然后自己修改的内容不断进行调整，调整后的内容提交到本地仓库，当完成一部分后，可以再提交至远程仓库，但是如果出现合并操作，出现异常，需要自己手动处理合并异常的文件。**
